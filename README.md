# FLH シフト希望提出システム

スタッフが各自のスマホからシフト希望を提出し、管理者が承認するシステム。
承認フローとGoogleカレンダー自動連携・Slack通知付き。

**アプリURL**: https://hirunegao.github.io/flh-shift/

## 主な機能

### スタッフ
- Googleアカウントでログイン（登録済みメールのみ）
- 月2回（前半1〜15日 / 後半16〜末日）のシフト希望提出
  - 締切: 前半→前月25日 / 後半→当月10日（締切後も警告付きで提出可）
- シフトパターンのワンタップ入力（スタンプ方式）＋時刻微調整、1日複数枠対応
- 未入力の日は「休み希望」扱い
- **完了ボタンで提出→ロック**。以後の変更は「変更リクエスト」→管理者承認が必要
- 提出時に自分のGoogleカレンダーへ「【未確定】出勤」を自動登録、承認で確定表記に更新
- 承認済みの全員のシフトを閲覧可能

### 管理者
- スタッフ単位で承認 / 差し戻し（理由付き）/ 直接修正
- グリッド表（縦=スタッフ、横=日付）＋カレンダービュー、日毎の人数集計
- 変更リクエストの承認（ロック解除）
- 共有カレンダー（全拠点まとめて1つ）に承認済みシフトを自動集約
- CSVダウンロード
- 拠点・シフトパターン・スタッフのマスタ管理（画面上で編集可）
- Slack通知＋メール通知、締切3日前・前日の自動リマインド

## 構成

| レイヤー | 技術 |
|---|---|
| フロントエンド | HTML/CSS/JavaScript（GitHub Pages） |
| バックエンド | Google Apps Script Web App（`gas/Code.gs`） |
| データベース | Googleスプレッドシート |
| 認証 | Google Identity Services |
| カレンダー連携 | Google Calendar API（OAuth 2.0） |
| 通知 | Slack Incoming Webhook＋Gmail |

運用コスト: **無料**（すべて無料枠内）

## ファイル構成

```

js/api.js         API通信層
js/app.js         認証・ルーティング・共通UI
js/staff.js       スタッフ画面（入力・みんなのシフト）
js/admin.js       管理者画面（承認・グリッド・マスタ管理）
gas/Code.gs       バックエンド（GASに貼り付け）
gas/appsscript.json GASマニフェスト
SETUP.md          セットアップ手順書
```
